1. Architecture Overview

Key Insight: Claude Code is built as a single-user, single-session CLI tool with sophisticated session management, telemetry, and state isolation. For your multi-user enterprise system, you'll need to replicate these patterns but with multi-tenancy awareness.

Core Architecture Pattern

Claude Code uses a centralized state + distributed execution model:

  • Bootstrap State (src/bootstrap/state.ts): Singleton global state - session ID, telemetry, counters
  • App State (src/state/): React-based UI state - messages, tasks, permissions
  • Services Layer: Analytics, MCP, API clients, policy limits
  • Tools Layer: Execution interface (Bash, File ops, Agents, MCP tools)
  • UI Layer: Ink-based terminal rendering (React reconciler for CLI)

High-Level Data Flow

1. Entry: cli.tsx → main.tsx → init.ts (bootstrap telemetry, auth, settings)
↓
2. Session Init: Generate SessionID → Setup telemetry sinks → Load policies
↓
3. UI Render: REPL.tsx (main loop) → Components (input, history, status)
↓
4. Query Processing: User input → query.ts → QueryEngine → API calls → Tool execution
↓
5. Telemetry: Every action logs to OpenTelemetry (metrics, traces, logs) → Exporters (OTLP, Datadog, BigQuery)

3. Telemetry System (Critical for Multi-User)

⚠️ Enterprise Critical: The telemetry system is your key to monitoring, training, and debugging multi-user sessions. Claude Code implements OpenTelemetry (OTEL) with three signals: Metrics, Traces, and Logs.

3.1 Telemetry Architecture

Telemetry Data Flow

Event Source: Tool execution, API calls, user interactions, errors
↓
Analytics API: logEvent() / logEventAsync() (src/services/analytics/index.ts)
↓
Sink Router: attachAnalyticsSink() → Routes to multiple backends
↓
Exporters:
  • Datadog (metrics/traces)
  • BigQuery (first-party events)
  • OTLP (OpenTelemetry Protocol)
  • Console (debugging)

3.4 Session Tracing (OpenTelemetry Spans)

Span Hierarchy: Claude Code uses a hierarchical span model to trace the full lifecycle of a user interaction.
// utils/telemetry/sessionTracing.ts // Span Types: // 1. interaction - User prompt → Claude response (root span) // 2. llm_request - Single API call to Claude // 3. tool - Tool execution (Bash, Read, Write, etc.) // 4. tool.blocked_on_user - Waiting for user approval // 5. tool.execution - Actual tool execution time // 6. hook - Hook execution (pre/post tool) // Example usage: export function startInteractionSpan(userPrompt: string): Span { const tracer = trace.getTracer('com.anthropic.claude_code.tracing'); const span = tracer.startSpan('interaction', { attributes: { 'span.type': 'interaction', 'user.prompt': truncateContent(userPrompt, 1000), 'session.id': getSessionId(), 'session.model': getMainLoopModel(), // ... base telemetry attributes } }); // Store in AsyncLocalStorage for nested spans interactionContext.enterWith({ span, startTime: Date.now() }); // Also create Perfetto span for Chrome trace viewer if (isPerfettoTracingEnabled()) { startInteractionPerfettoSpan(userPrompt); } return span; } export function startLLMRequestSpan(messages: APIMessage[]): Span { const parentSpan = interactionContext.getStore()?.span; const tracer = trace.getTracer('com.anthropic.claude_code.tracing'); return tracer.startSpan('llm_request', { attributes: { 'span.type': 'llm_request', 'llm.request.message_count': messages.length, 'llm.request.model': getMainLoopModel(), // ... PII-safe attributes only } }, otelContext.active()); }

3.6 PII Protection

Security Pattern: All telemetry uses type markers to enforce PII safety:
  • AnalyticsMetadata_I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS - General analytics
  • AnalyticsMetadata_I_VERIFIED_THIS_IS_PII_TAGGED - Routed to PII-safe storage (BigQuery with access controls)
// services/analytics/index.ts export type AnalyticsMetadata_I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS = never; // Forces explicit type casting - developer must verify safety logEvent('event_name', { tool_name: sanitizedName as AnalyticsMetadata_I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS, // NOT ALLOWED: user_input: rawInput (would be PII) }); // PII-tagged values use _PROTO_ prefix and route to protected storage logEvent('event_name', { _PROTO_user_id: userId as AnalyticsMetadata_I_VERIFIED_THIS_IS_PII_TAGGED, _PROTO_org_id: orgId as AnalyticsMetadata_I_VERIFIED_THIS_IS_PII_TAGGED, });

// utils/sessionStorage.ts // Session directory: ~/.claude/sessions/ // Files per session: // - {sessionId}.jsonl - Main conversation log // - {sessionId}.metadata.json - Session metadata // - {sessionId}.title - Human-readable title export async function saveMessageToSession( sessionId: SessionId, message: MessageType ): Promise { const sessionPath = getSessionLogPath(sessionId); const entry = { type: 'message', role: message.role, content: message.content, timestamp: Date.now(), }; // Append to JSONL (newline-delimited JSON) await appendFile(sessionPath, JSON.stringify(entry) + '\n'); } export async function loadTranscriptFromFile( sessionId: SessionId ): Promise { const sessionPath = getSessionLogPath(sessionId); const lines = (await readFile(sessionPath, 'utf-8')).split('\n'); return lines .filter(line => line.trim()) .map(line => JSON.parse(line)) .filter(entry => entry.type === 'message') .map(entry => ({ role: entry.role, content: entry.content, })); }

4.5 Concurrent Session Management

⚠️ Single-User Limitation: Claude Code supports multiple concurrent sessions per user but lacks multi-tenant isolation. For enterprise, you'll need to add:
  • User/Organization ID to session metadata
  • Session isolation enforcement (prevent cross-user access)
  • Resource quotas per user/org
  • Session affinity (route user to their sessions)
// utils/concurrentSessions.ts - Session registry const activeSessions = new Map(); export function registerSession( sessionId: SessionId, metadata: SessionMetadata ): void { activeSessions.set(sessionId, { ...metadata, registeredAt: Date.now(), }); } export function countConcurrentSessions(): number { return activeSessions.size; } // For enterprise multi-user, extend to: export function getSessionsByUser(userId: string): SessionId[] { return Array.from(activeSessions.entries()) .filter(([_, meta]) => meta.userId === userId) .map(([id, _]) => id); }
