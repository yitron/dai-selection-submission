Execution Path from CLI Command

// src/entrypoints/cli.tsx (27,674 tokens) async function main() { const args = process.argv.slice(2); // FAST PATH 1: --version (zero imports) if (args[0] === '--version') { console.log(`${MACRO.VERSION} (Claude Code)`); return; } // FAST PATH 2: --dump-system-prompt (ant-only) if (args[0] === '--dump-system-prompt') { // Dynamic import to avoid loading full app return; } // FAST PATH 3: Bridge mode (IDE extension) if (args[0] === 'remote-control' || args[0] === 'bridge') { const { bridgeMain } = await import('../bridge/bridgeMain.js'); await bridgeMain(args.slice(1)); return; } // FAST PATH 4: Daemon mode if (args[0] === 'daemon') { const { daemonMain } = await import('../daemon/main.js'); await daemonMain(args.slice(1)); return; } // FAST PATH 5: Background sessions (ps, logs, attach, kill) if (args[0] === 'ps' || args[0] === 'logs') { // ...handle background session management return; } // DEFAULT: Load full application (main.tsx) const { main: mainFunction } = await import('../main.js'); await mainFunction(); }

This is where the REAL application begins for interactive sessions

⚡ Why Fast Paths?

Performance Optimization: Fast paths avoid loading unnecessary modules
Command Module Loading Startup Time
claude --version Zero modules (just MACRO.VERSION) ~1ms
claude bridge Only bridge module (~115KB) ~50ms
claude (interactive) Full application (main.tsx) ~200ms

🔧 Execution Modes

1. Interactive REPL (Default)

$ claude # Loads main.tsx → REPL.tsx # Full conversation loop with UI rendering

2. Bridge Mode (IDE Extension)

$ claude remote-control # Connects local machine to claude.ai bridge # Polls for work assignments every 2 seconds # Spawns child CLI processes for each session

3. Daemon Mode (Long-Running Supervisor)

$ claude daemon start # Runs supervisor process that manages workers # Different worker types: assistant, bridge, etc.

4. Background Sessions

$ claude ps # List running sessions $ claude logs # View session logs $ claude attach # Attach to session $ claude kill # Kill session

5. MCP Server Mode

$ claude mcp serve # Runs as Model Context Protocol server # Exposes Claude Code tools to other MCP clients

6. Direct Connect

$ claude open # Connect to a direct-connect server # Uses WebSocket for real-time communication

7. Assistant Mode (KAIROS)

$ claude assistant [session-id] # Connect to remote CCR session # View-only mode (no sending commands)

📦 Module Loading Strategy

Design Pattern: Lazy loading with feature flags for dead code elimination
// Feature-gated imports (compiled out if feature disabled) import { feature } from 'bun:bundle'; const coordinatorModule = feature('COORDINATOR_MODE') ? require('./coordinator/coordinatorMode.js') : null; const assistantModule = feature('KAIROS') ? require('./assistant/index.js') : null; const daemonWorker = feature('DAEMON') ? require('./daemon/workerRegistry.js') : null; // Dynamic imports for heavy modules if (args[0] === 'bridge') { const { bridgeMain } = await import('../bridge/bridgeMain.js'); // Only loads when actually needed }

Benefits for Enterprise

  • ✅ Smaller bundles: Unused features are stripped at build time
  • ✅ Faster startup: Only load what's needed for the current mode
  • ✅ Easy A/B testing: Enable/disable features without code changes
  • ✅ Gradual rollouts: Feature flags control access
