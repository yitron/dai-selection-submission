🏢 Enterprise Adaptation:

For multi-user systems, you'll need to:

  • Isolate query state per user session (avoid shared global state)
  • Implement user-level rate limiting in QueryEngine
  • Add session affinity routing (route user to same agent instance)
  • Implement request queueing for fairness across users
  • Add circuit breakers for failing user sessions

2. Harness Toolkit

Tool system providing agent capabilities: file ops, bash, web search, agents, MCP

Tool System Architecture

Design Pattern: Claude Code implements tools as independent modules with a common interface. Each tool is responsible for:
  • Input validation (Zod schemas)
  • Permission checks
  • Execution logic
  • Result formatting
  • Error handling

🏢 Enterprise Toolkit Adaptations:
  • Tool Isolation: Run tools in user-specific containers/sandboxes
  • Resource Limits: CPU/memory/time limits per user/org
  • Tool Quotas: Track tool usage per user (e.g., max 100 Bash calls/hour)
  • Tool Allowlists: Per-org tool restrictions (disable BashTool for some orgs)
  • Audit Logging: Log all tool executions with user/org context for compliance
  • Tool Result Caching: Cache deterministic tool results (e.g., file reads) per user

3. Context & Memory Management

Long-term memory, conversation history, context injection, file tracking

Memory System Architecture

Memory Layers in Claude Code:
  • Session Memory: Current conversation context (messages array)
  • Long-term Memory: MEMORY.md file with structured memories
  • File History: Recently accessed files for context
  • System Context: Git status, working directory, environment
  • CLAUDE.md: Project-specific instructions
