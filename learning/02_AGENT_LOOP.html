🔄 Recursive Query Pattern

Key Insight: The agent loop is recursive. When tools are invoked, the loop continues with tool results added to the conversation.
// Simplified recursive pattern async function query(messages: Message[]): Promise { // 1. Call LLM const response = await llm.query(messages); // 2. Check if tools were used const toolUses = extractToolUses(response); if (toolUses.length === 0) { // Base case: No tools, return final response return response; } // 3. Execute all tools const toolResults = await Promise.all( toolUses.map(tool => executeTool(tool)) ); // 4. Recursive case: Add results and continue return query([ ...messages, response, // Assistant message with tool_use blocks ...toolResults, // Tool result messages ]); }
Loop Termination: The recursive loop terminates when:
  • The LLM returns a response with no tool uses
  • Max turns limit is reached
  • User cancels (Ctrl+C)
  • An error occurs and retry logic exhausts

Stream Events: The API returns responses as a stream of events, allowing real-time UI updates.
// QueryEngine.ts - Stream event types for await (const event of response) { switch (event.type) { case 'message_start': // Message begins break; case 'content_block_start': // New content block (text or tool_use) if (event.content_block.type === 'tool_use') { // Tool invocation starting } break; case 'content_block_delta': // Incremental content (streaming text) if (event.delta.type === 'text_delta') { appendText(event.delta.text); } break; case 'content_block_stop': // Content block complete break; case 'message_delta': // Message metadata updates (stop_reason, usage) break; case 'message_stop': // Message complete break; } }

🏢 Enterprise Adaptations

Multi-User Query Management:

For enterprise multi-tenant systems, you'll need to adapt the agent loop:

1. State Isolation

  • Problem: Global STATE singleton is shared across all users
  • Solution: Create per-user context objects passed through query chain
// Enterprise pattern: Per-user context type UserContext = { userId: string; orgId: string; sessionId: string; permissions: Permission[]; quotas: ResourceQuotas; }; async function query( messages: Message[], context: UserContext ): Promise { // All operations use context instead of global STATE }

2. Rate Limiting

  • Implement per-user/org rate limits in QueryEngine
  • Track API calls, tokens, and tool executions per user
  • Queue requests when limits are reached

3. Session Affinity

  • Route user to same agent instance for conversation continuity
  • Use sticky sessions or distributed cache for state
  • Handle instance failures with session migration

4. Request Queueing

  • Fair scheduling across users (prevent one user monopolizing)
  • Priority queues for premium users/orgs
  • Circuit breakers for failing user sessions

5. Observability

  • Tag all telemetry with user_id and org_id
  • Track per-user costs and usage metrics
  • Alert on anomalous user behavior
